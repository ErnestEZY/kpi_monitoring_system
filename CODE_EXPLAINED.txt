This document explains every PHP file, what each function does,

================================================================================
  TECHNICAL TERMS GLOSSARY
================================================================================

PDO (PHP Data Objects)
  A PHP extension that provides a consistent interface for accessing
  databases. Instead of writing MySQL-specific code, you write PDO code
  that works with any database. More importantly, PDO supports PREPARED
  STATEMENTS which prevent SQL injection.

Prepared Statement
  A way of writing SQL queries where user input is kept SEPARATE from
  the SQL code. The database receives the query structure first, then
  the data separately. This means a user cannot inject malicious SQL
  because their input is always treated as data, never as code.
  Example WITHOUT prepared statement (DANGEROUS):
    "SELECT * FROM staff WHERE id = " . $_GET['id']
    If id = "1 OR 1=1", this returns ALL staff — SQL injection!
  Example WITH prepared statement (SAFE):
    $stmt = $pdo->prepare("SELECT * FROM staff WHERE id = ?");
    $stmt->execute([$_GET['id']]);
    The ? is a placeholder. The value is bound separately and cannot
    change the structure of the SQL query.

Session
  A way to store data on the SERVER between page requests. When a user
  logs in, PHP creates a session file on the server and gives the browser
  a session ID cookie. On every subsequent request, the browser sends
  that cookie, and PHP loads the matching session data. This is how the
  system "remembers" who is logged in across multiple pages.

AJAX (Asynchronous JavaScript and XML)
  A technique where JavaScript sends a request to the server in the
  BACKGROUND without reloading the page. The server responds with data
  (usually JSON), and JavaScript updates only the relevant part of the
  page. Used extensively in this system for saving scores, loading charts,
  and the comment assistant.

JSON (JavaScript Object Notation)
  A lightweight data format used to send data between the PHP backend
  and JavaScript frontend. Looks like: {"success": true, "data": [...]}
  PHP uses json_encode() to create it, JavaScript uses JSON.parse() or
  response.json() to read it.

Transaction
  A group of database operations that either ALL succeed or ALL fail
  together. Used in save_scores: if saving 21 KPI scores and the 15th
  one fails, the transaction rolls back all 15 already saved. This
  ensures the database never has partial/incomplete data.

Foreign Key
  A column in one table that references the primary key of another table.
  Enforces referential integrity — you cannot save a score for a staff
  member that doesn't exist. The database itself enforces this rule.

CASCADE / RESTRICT / SET NULL
  Rules that define what happens to related records when a parent record
  is deleted:
  CASCADE  — delete the child records too (delete staff → delete their scores)
  RESTRICT — prevent deletion if child records exist (cannot delete a KPI
             that has scores attached)
  SET NULL — keep child records but set the FK column to NULL (delete a
             supervisor → their comments remain, supervisor_id becomes NULL)

XSS (Cross-Site Scripting)
  An attack where malicious JavaScript is injected into a web page.
  Prevented by htmlspecialchars() which converts < > " & into safe
  HTML entities so they display as text, not execute as code.

Singleton Pattern
  A design pattern that ensures only ONE instance of something is created.
  Used in getDBConnection() — the database connection is created once and
  reused for the rest of the request, saving resources.

OLS (Ordinary Least Squares)
  The standard mathematical method for fitting a straight line through
  data points. Used in predictPerformance() to find the line that
  minimises the sum of squared differences between actual and predicted
  values. The formula gives the slope (m) and intercept (b) of the line.

R² (R-Squared / Coefficient of Determination)
  A statistical measure of how well a regression line fits the data.
  R² = 1 means perfect fit. R² = 0 means the line explains nothing.
  Used to calculate confidence in predictions: higher R² = higher confidence.

MAE (Mean Absolute Error)
  The average of the absolute differences between predicted and actual
  values. Used in TensorFlow.js to measure how well the neural network
  fits the training data. Lower MAE = better fit = higher confidence.

ReLU (Rectified Linear Unit)
  An activation function used in neural networks: f(x) = max(0, x).
  It outputs 0 for negative inputs and x for positive inputs.
  This allows the network to learn non-linear patterns (curves, not
  just straight lines). Used in the hidden layers of the TF.js model.

Normalisation (data)
  Scaling numbers to a standard range (usually 0–1) before feeding them
  into a neural network. This makes training faster and more stable.
  In TF.js: year_index / (n-1) and score / 100.

3NF (Third Normal Form)
  A database design standard that eliminates redundancy:
  1NF: No repeating groups (each cell has one value)
  2NF: Every non-key column depends on the WHOLE primary key
  3NF: No transitive dependencies (A→B→C split into separate tables)
  Example: storing department_name in kpi_scores would violate 3NF
  because it depends on staff_id, not on the score itself.

================================================================================
  FILE: config/database.php
================================================================================

PURPOSE:
  The single source of truth for database connection settings.
  Every PHP file that needs the database calls getDBConnection() from here.
  This means if the database password changes, you only update ONE file.

CONSTANTS (DB_HOST, DB_NAME, DB_USER, DB_PASS, DB_CHARSET):
  PHP constants defined with const. Unlike variables ($var), constants
  cannot be changed after definition. Using const instead of define()
  is the modern PHP approach and slightly more efficient.

FUNCTION: getDBConnection(): PDO
  Returns a PDO database connection object.

  static $pdo = null
    The static keyword means this variable persists between calls to
    the function within the same request. The first call creates the
    connection; subsequent calls return the same connection object.
    This is the Singleton pattern — prevents creating multiple
    connections to the database unnecessarily.

  sprintf() for DSN
    DSN = Data Source Name. A string that tells PDO how to connect:
    "mysql:host=localhost;dbname=kpi_system;charset=utf8mb4"
    Using sprintf() instead of string concatenation is cleaner and
    easier to read.

  PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION
    Tells PDO to throw a PDOException when a query fails, instead of
    silently returning false. This means errors are caught by try-catch
    blocks and handled properly.

  PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC
    When fetching rows, return them as associative arrays
    (e.g. $row['name']) instead of numeric arrays ($row[0]).
    Makes code more readable and less error-prone.

  PDO::ATTR_EMULATE_PREPARES => false
    Forces PDO to use REAL prepared statements at the database level,
    not emulated ones. Real prepared statements are more secure because
    the SQL and data are truly separated at the protocol level.

  catch (PDOException $e)
    If connection fails: logs the real error to the server error log
    (not shown to users — security), sets HTTP 500 status, and returns
    a safe JSON error message. The real error is never exposed to the
    browser.

================================================================================
  FILE: includes/auth.php
================================================================================

PURPOSE:
  Centralises all authentication logic. Every protected page calls
  requireLogin() at the very top. If the check fails, the user is
  redirected before any page content is generated.

FUNCTION: requireLogin(): void
  Checks session_status() first — if no session is running, starts one.
  Then checks $_SESSION['supervisor_id']. If not set, the user is not
  logged in, so redirect to login.php and exit() immediately.
  exit() is critical — without it, PHP would continue executing the
  rest of the page even after the redirect header is sent.

FUNCTION: requireLoginApi(): void
  Same logic but for API endpoints. Instead of redirecting (which would
  return HTML to a JavaScript fetch() call), it returns a JSON error
  with HTTP status 401 (Unauthorized). JavaScript can then handle this
  gracefully.

FUNCTION: isLoggedIn(): bool
  Returns true/false. Used for conditional checks in pages that need
  to know login status without enforcing it.

FUNCTION: getSupervisorId(): ?int
  The ?int return type means it can return either an integer or null.
  Casts the session value to int to ensure type safety — session values
  are strings by default, and using a string as a database ID could
  cause unexpected behaviour.

FUNCTION: staffPhotoTag(): ?string
  Generates an HTML img tag for a staff photo. Returns null if no photo
  is set, allowing pages to use the initials avatar as fallback.
  htmlspecialchars() on the path prevents XSS if a malicious filename
  were somehow stored in the database.

================================================================================
  FILE: includes/KPICalculator.php
================================================================================

PURPOSE:
  The core business logic class. Contains all calculations, classifications,
  and analysis methods. By putting all this logic in one class, any page
  that needs KPI calculations just creates a KPICalculator object and
  calls the appropriate method. This is the principle of DRY (Don't
  Repeat Yourself) — the formula is written once, used everywhere.

CLASS STRUCTURE:
  private $pdo — stores the database connection
  __construct($pdo) — receives the connection when the object is created
  All methods are public — accessible from outside the class

METHOD: calculateOverallScore($staff_id, $year)
  The most important method in the system.

  SQL JOIN explanation:
    SELECT from kpi_master (all 21 KPIs) LEFT JOIN kpi_scores
    LEFT JOIN means: include ALL KPIs even if no score exists for this
    staff/year. COALESCE(ks.score, 0) returns 0 if no score found.
    This ensures the method always returns 21 rows, one per KPI.

  Calculation:
    For each KPI with a score > 0:
      total_weighted_score += weighted_score
    weighted_score is already stored in the database as:
      (score / 5) × (weight_percentage / 100)
    So total_weighted_score is in range 0.0 to 1.0
    overall_score = total_weighted_score × 100 (converts to percentage)

  Returns:
    overall_score — the final percentage (0–100)
    category_scores — array of individual KPI data for charts
    has_data — boolean, false if no scores exist for this year

METHOD: getPerformanceTrend($staff_id)
  Gets all years the staff has data for, then calls
  calculateOverallScore() for each year.
  Calculates change = current_score - previous_score
  change_percentage = (change / previous_score) × 100
  determineTrend(): >2 = improving, <-2 = declining, else stable
  Returns chronological array used for line charts.

METHOD: classifyPerformance($score)
  Simple if-else chain returning a text label.
  Thresholds: 85/75/65/50 — these are the business rules defined
  by the system requirements.

METHOD: getPerformanceClassification($score)
  Extended version returning label + Bootstrap badge class + hex colour.
  The badge class (success/info/warning/danger) maps directly to
  Bootstrap's colour system for consistent UI styling.

METHOD: getAtRiskStaff($current_year)
  FIXED VERSION — uses the selected year directly.
  Previous bug: used end($trend) which always returned the most recent
  year in the database regardless of the year filter.
  Fix: calls calculateOverallScore($staff_id, $current_year) directly.
  Two criteria:
    HIGH RISK: score < 60% in the selected year alone
    MEDIUM RISK: score < 70% in BOTH selected year AND previous year
  The previous year check (current_year - 1) identifies staff who have
  been underperforming consistently, not just having a bad year.

METHOD: getTopPerformers($year, $threshold = 85)
  Loops all active staff, calls calculateOverallScore() for each.
  Default threshold is 85% (Top Performer classification).
  usort() sorts the results by score descending using the spaceship
  operator (<=>): returns -1, 0, or 1 for comparison.

METHOD: getCategoryAverages($year)
  Groups KPI scores by kpi_group (category) using SQL GROUP BY.
  AVG() calculates the average score per category across all staff.
  Multiplies by 20 to convert from 1–5 scale to 0–100%:
    (avg_score / 5) × 100 = avg_score × 20
  Used for the radar chart in analytics.

METHOD: generateNarrative($staff_id)
  Builds a plain-English paragraph from data.
  Analyses: current classification, year-on-year change, multi-year
  trend, category strengths (≥85%) and weaknesses (<70%).
  Uses implode() to join sentences into a paragraph.
  This is rule-based narrative generation — not AI, but structured
  text templates filled with real data values.

METHOD: compareToTeamAverage($staff_id, $year)
  Subquery calculates team average:
    SELECT AVG(total_score) FROM (SELECT SUM(weighted_score) ...)
  The inner query sums weighted scores per staff member.
  The outer query averages those sums.
  Multiplies by 100 to convert to percentage.

METHOD: predictPerformance($staff_id)
  Implements Ordinary Least Squares (OLS) linear regression.
  x = year index (0, 1, 2, ...), y = overall score
  Formulas:
    m = (n·Σxy − Σx·Σy) / (n·Σx² − (Σx)²)   [slope]
    b = (Σy − m·Σx) / n                         [intercept]
    predicted = m × n + b                        [next year]
  R² = 1 − (SS_residual / SS_total)
    SS_total = Σ(y − mean_y)²
    SS_residual = Σ(y − predicted_y)²
  Confidence = clamp(R² × 100, 65, 95)
  Requires minimum 3 data points to avoid overfitting.

METHOD: getSmartTrainingRecommendations($staff_id, $year)
  Identifies skill gaps by comparing category scores to thresholds.
  For each of 6 training programs, calculates a match score based on
  how much the program's focus areas overlap with the staff's gaps.
  Higher gap severity = higher match score for relevant programs.
  Prerequisites check: Leadership program requires score ≥85%.
  Returns top 5 sorted by match score descending.

================================================================================
  FILE: api/kpi_api.php
================================================================================

PURPOSE:
  Handles AJAX requests from JavaScript. All responses are JSON.
  Uses a switch statement to route to the correct function based on
  the ?action= URL parameter.

requireLoginApi() at the top ensures unauthenticated requests get
a 401 JSON response, not a redirect to the login page.

FUNCTION: getStatistics($pdo)
  Returns 4 numbers for the analytics summary cards.
  Uses SUM(weighted_score) to get final scores, then counts how many
  are above/below thresholds. HAVING clause filters out staff with
  no scores (NULL).

FUNCTION: getStaffPerformance($pdo)
  Uses CASE WHEN inside SUM() to split scores by section:
    SUM(CASE WHEN section_number = 1 THEN weighted_score ELSE 0 END)
  This is a pivot operation — turning rows into columns.
  Returns Core Competencies score and KPI Achievement score separately.

FUNCTION: getStaffComment($pdo) / saveComment($pdo)
  Read and write supervisor comments.
  saveComment uses INSERT ... ON DUPLICATE KEY UPDATE:
    If a comment for this staff/year already exists → UPDATE it
    If not → INSERT a new one
  This is more efficient than checking first with SELECT then deciding.

FUNCTION: saveScores($pdo)  ← MOST IMPORTANT
  Accepts JSON body (not form data) — reads with file_get_contents('php://input')
  and json_decode().

  Batch weight loading (N+1 fix):
    BEFORE: one SELECT per KPI inside the loop = 21 queries
    AFTER: one SELECT for ALL KPIs before the loop = 1 query
    implode(',', array_fill(0, count($scores), '?')) creates the
    right number of ? placeholders for the IN() clause.
    fetchAll(PDO::FETCH_KEY_PAIR) returns [kpi_code => weight] directly.

  Transaction:
    beginTransaction() → all INSERTs → commit() or rollBack()
    If any INSERT fails, rollBack() undoes everything already saved.

  ON DUPLICATE KEY UPDATE:
    The unique key (staff_id, kpi_code, evaluation_year) means the same
    staff cannot have two scores for the same KPI in the same year.
    If a duplicate is attempted, UPDATE the existing row instead.

  Returns overall_score calculated server-side after saving, so the
  UI shows the confirmed value, not just the client-side estimate.

PURPOSE:
  A read-only API that exposes KPICalculator methods to JavaScript.

================================================================================
  FILE: api/innovative_features_api.php
================================================================================

PURPOSE:
  Handles the 4 innovative features. All endpoints require login.

FUNCTION: getAllTrends($pdo)  ← KEY FOR TENSORFLOW.JS
  Queries kpi_scores grouped by staff_id and evaluation_year.
  SUM(weighted_score) × 100 gives the overall percentage per year.
  Groups results by staff_id in PHP to build the structure:
    {staff_id, staff_name, years: [2022,2023,...], scores: [72,75,...]}
  This is exactly what TensorFlow.js needs to train on.

FUNCTION: getPredictiveAlerts($pdo)
  PHP-based simpler version (not the TF.js one).
  predictFutureScore() adds a small trend factor based on current score.
  assessRisk() classifies based on predicted score and rate of decline.
  Used as a fallback if TF.js is not available.

FUNCTION: getTrainingRecommendations($pdo)
  Calls getSmartTrainingRecommendations() for all staff below score 4.0.
  Formats results for display: skill gaps as badges, programs as cards.
  Calculates summary statistics: staff count, program count, critical gaps.

FUNCTION: getGamificationData($pdo)
  Calculates points for each staff member using the points formula.
  Assigns levels based on point thresholds.
  Returns leaderboard sorted by points descending.

FUNCTION: getPeerComparison($pdo)
  Finds staff with similar scores (within ±10%).
  Calculates similarity = 100 - |score_difference|.
  Returns comparison data including category-level scores for radar chart.

FUNCTION: detect_anomalies (inline in switch)
  Loops through trend data comparing consecutive years.
  Spike: change >= 5 (sudden improvement)
  Drop: change <= -5 (sudden decline)
  Decline: 3+ consecutive years of decrease
  Returns array of anomaly objects with type, year, description.

================================================================================
  FILE: auth/login.php
================================================================================

PURPOSE:
  Handles both displaying the login form (GET) and processing it (POST).
  PHP checks $_SERVER['REQUEST_METHOD'] to know which to do.

POST PROCESSING:
  1. trim() removes whitespace from username (prevents " admin" matching)
  2. Queries supervisors table WHERE username = ? AND status = 'Active'
     The status check prevents disabled accounts from logging in.
  3. Password comparison: $password === $supervisor['password']
     Triple equals (===) checks value AND type (strict comparison).
     Plain text as per project requirements.
  4. On success: stores 5 values in $_SESSION, records login_time,
     updates last_login in database, redirects.
  5. On failure: sets $error variable, page re-renders with error shown.

SESSION VALUES STORED:
  supervisor_id       — used by requireLogin() to verify authentication
  supervisor_name     — displayed in navbar
  supervisor_email    — available for future use
  supervisor_username — available for future use
  login_time          — displayed in navbar as "Logged in: Apr 07, 2026"

================================================================================
  FILE: auth/logout.php
================================================================================

THREE-STEP LOGOUT:
  1. $_SESSION = [] — clears all session data (variables)
  2. setcookie() with past expiry — tells browser to delete the cookie
  3. session_destroy() — removes the session file from the server
  All three steps are needed for a complete logout. Just calling
  session_destroy() without clearing variables can leave data accessible
  in the same request.

================================================================================
  FILE: supervisor/predictive_alerts_new.php
================================================================================

PURPOSE:
  The TensorFlow.js predictive alerts page. Most of the logic is
  JavaScript, but the PHP part loads the page structure and includes
  the navbar/sidebar.

TENSORFLOW.JS EXPLAINED:
  TensorFlow.js is loaded from CDN with a single script tag.
  It runs entirely in the browser — no Python, no server ML needed.

  tf.ready()
    Returns a Promise that resolves when TF.js has initialised and
    chosen a backend (WebGL for GPU, or WASM/CPU as fallback).
    The backend is shown in the status indicator.

  trainModel(years, scores)
    1. Normalise: xRaw = year_index / (n-1), yRaw = score / 100
       Normalisation keeps values in [0,1] which makes gradient descent
       (the training algorithm) converge faster and more stably.
    2. tf.tensor2d() converts JavaScript arrays to TensorFlow tensors
       (multi-dimensional arrays optimised for GPU computation).
    3. tf.sequential() creates a model where layers are stacked linearly.
    4. tf.layers.dense() creates a fully-connected layer where every
       neuron connects to every neuron in the next layer.
    5. ReLU activation: f(x) = max(0, x) — allows non-linear learning.
       Linear activation on output: allows any real number as output.
    6. model.compile() sets the optimiser and loss function:
       Adam optimiser — adaptive learning rate, works well in practice
       MSE loss — penalises large errors more than small ones
    7. model.fit() trains for 200 epochs (passes through the data).
       verbose: 0 means no console output during training.
    8. model.predict() runs the trained model on the next year's index.
    9. .dataSync() extracts the result from the tensor to a JS number.
    10. Tensors are disposed after use to free GPU memory.

  classifyRisk(current, predicted)
    Pure score-based classification matching the threshold bar:
    < 50% = Critical, < 65% = High, < 75% = Medium, >= 75% = Low
    No drop-based conditions — classification is purely on predicted score.

  renderAlerts(results)
    Builds HTML strings for each alert card and sets innerHTML.
    lowSection() creates the collapsed Bootstrap accordion for
    Low/Stable staff — hidden by default, expandable on click.

================================================================================
  FILE: supervisor/analytics.php + assets/js/analytics.js
================================================================================

PURPOSE:
  The interactive analytics dashboard. PHP renders the page shell;
  JavaScript loads all data and charts via AJAX.

PARALLEL LOADING (Promise.allSettled):
  loadAnalytics() calls all 7 loaders simultaneously using
  Promise.allSettled(). Unlike Promise.all(), allSettled() continues
  even if one loader fails — other charts still render.
  This is faster than sequential loading (all requests in parallel).

CHART STORYTELLING:
  Each chart loader generates a dynamic insight after rendering.
  The insight text changes based on actual data values:
    - Distribution: checks if >30% are top performers
    - Department: calculates gap between highest and lowest
    - Trend: compares first and last year averages
    - Category: identifies strongest and weakest categories
    - BoxPlot: analyses median and spread

filterStaffData(data):
  Filters the staff array client-side (no server request needed).
  Uses data.department (string) not data.department_id (was a bug —
  the field is a string, not a numeric ID).
