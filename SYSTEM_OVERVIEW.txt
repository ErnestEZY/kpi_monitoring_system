================================================================================
  SYSTEM PURPOSE
================================================================================

This system was built to solve a real workplace problem: supervisors in a
retail environment need a structured, data-driven way to monitor how their
sales assistants are performing across multiple areas of their job.

Without a system like this, supervisors would rely on spreadsheets or memory,
making it hard to spot trends, compare staff fairly, or justify training
decisions. This system replaces that with:

  - A structured 21-KPI scoring framework (1–5 scale, weighted)
  - Multi-year historical tracking (2022–2026)
  - Automatic performance classification and trend detection
  - AI-powered predictions using TensorFlow.js (neural network)
  - Smart training recommendations matched to individual skill gaps
  - Anomaly detection to flag unusual performance changes
  - Peer comparison to benchmark staff against similar colleagues
  - Gamification to motivate staff through points and badges

The system is supervisor-only. There is no sales assistant login — supervisors
enter scores on behalf of their team and view all analytics.

================================================================================
  TECHNOLOGY STACK
================================================================================

FRONTEND (what the user sees in the browser):
  Bootstrap 5.3.0     — Responsive layout framework. Makes the UI look
                        professional and work on different screen sizes
                        without writing custom CSS for every element.
  Chart.js 4.4.0      — JavaScript charting library. Used for doughnut,
                        line, bar, and radar charts throughout the system.
  jQuery 3.7.1        — Simplifies DOM manipulation and AJAX calls.
                        Used in training recommendations and peer comparison.
  Alpine.js 3.x       — Lightweight reactive framework. Used specifically
                        for the Smart Comment Assistant modal on staff profiles.
                        Allows two-way data binding without a full framework.
  SweetAlert2 11.x    — Replaces the browser's default alert() with styled
                        modals. Used for confirmations, loading states, toasts.
  Bootstrap Icons     — Icon library. Used throughout for visual clarity.
  TensorFlow.js 4.17  — Google's machine learning library ported to JavaScript.
                        Runs a neural network INSIDE the browser — no server
                        needed. Used exclusively for Predictive Alerts.

BACKEND (server-side logic):
  PHP 7.4+            — Server-side scripting language. Handles all database
                        queries, business logic, and HTML generation.
  PDO                 — PHP Data Objects. A database abstraction layer that
                        supports prepared statements to prevent SQL injection.
  MySQL / MariaDB     — Relational database. Stores all staff, KPI, and
                        score data. MariaDB is the version bundled with XAMPP.

ARCHITECTURE:
  The system follows a 3-tier architecture:
    1. Presentation Layer  — PHP/HTML pages in supervisor/
    2. Business Logic      — KPICalculator.php class + API endpoints
    3. Data Layer          — MySQL database accessed via PDO

  API endpoints (api/ folder) return JSON and are called by JavaScript
  using the Fetch API or jQuery AJAX. This separates data from display
  and allows charts to update without full page reloads.

SECURITY MEASURES:
  - PDO Prepared Statements  — User input is never directly placed into
                               SQL strings. Parameters are bound separately,
                               so SQL injection is not possible.
  - htmlspecialchars()       — All user-supplied text is escaped before
                               being output to HTML, preventing XSS attacks.
  - Session Authentication   — Every protected page checks for a valid
                               session before rendering. No page is accessible
                               without logging in first.
  - .htaccess Protection     — The config/ folder is blocked from direct
                               browser access, protecting DB credentials.

================================================================================
  DATABASE STRUCTURE (5 Tables, 3NF Normalised)
================================================================================

Third Normal Form (3NF) means:
  - No duplicate data (each piece of information stored once)
  - Every column depends only on the primary key
  - No transitive dependencies (A → B → C is split into separate tables)

TABLE: supervisors
  Stores supervisor login accounts.
  Columns: supervisor_id (PK), username, password, full_name, email,
           status, last_login, created_at, updated_at
  Sample data: 3 supervisors (admin, manager1, manager2)

TABLE: staff
  Stores sales assistant profiles.
  Columns: staff_id (PK), staff_code, name, email, department, position,
           status, hire_date, photo, created_at
  Sample data: 13 active staff members across Electronics, Fashion,
               Home Goods departments
  NOTE: photo column stores just the filename (e.g. "Alice_Johnson.jpg")
        not the full path — this is normalisation in practice.

TABLE: kpi_master
  Stores the definition of all 21 KPIs. This is a lookup/reference table.
  Columns: kpi_id (PK), kpi_code, section, kpi_group, kpi_description,
           weight_percentage, section_number, display_order, created_at
  Why separate? If KPI descriptions or weights change, you update one row
  here instead of thousands of score rows.

TABLE: kpi_scores
  Stores the actual scores entered by supervisors.
  Columns: score_id (PK), staff_id (FK), kpi_code (FK), evaluation_date,
           evaluation_year, score (1–5), weighted_score, created_at, updated_at
  Foreign Keys:
    staff_id → staff(staff_id) ON DELETE CASCADE
    kpi_code → kpi_master(kpi_code) ON DELETE RESTRICT
  Unique Key: (staff_id, kpi_code, evaluation_year)
    This prevents duplicate scores for the same staff/KPI/year.
    ON DUPLICATE KEY UPDATE means re-submitting updates instead of errors.
  Sample data: 742 score rows across 2022–2026

TABLE: staff_comments
  Stores supervisor feedback and training recommendations per staff per year.
  Columns: comment_id (PK), staff_id (FK), evaluation_year, supervisor_comment,
           training_recommendation, supervisor_id (FK), created_at, updated_at
  Unique Key: (staff_id, evaluation_year) — one comment set per year per staff
  Sample data: 14 comment records

FOREIGN KEY RELATIONSHIPS:
  kpi_scores.staff_id      → staff.staff_id         (CASCADE delete)
  kpi_scores.kpi_code      → kpi_master.kpi_code    (RESTRICT delete)
  staff_comments.staff_id  → staff.staff_id         (CASCADE delete)
  staff_comments.supervisor_id → supervisors.supervisor_id (SET NULL)

  CASCADE: if a staff member is deleted, their scores are deleted too.
  RESTRICT: you cannot delete a KPI that has scores attached to it.
  SET NULL: if a supervisor is deleted, their comments remain but
            supervisor_id becomes NULL (comments are preserved).

SCORING SCALE:
  5 = Excellent    (Outstanding performance)
  4 = Good         (Above average)
  3 = Satisfactory (Meets expectations)
  2 = Poor         (Below average)
  1 = Very Poor    (Unsatisfactory)

CALCULATION FORMULA:
  weighted_score  = (score / 5) × (weight_percentage / 100)
  overall_score % = SUM(all 21 weighted_scores) × 100

  Example: KPI1 score=4, weight=5%
    weighted_score = (4/5) × (5/100) = 0.04
    If all 21 KPIs scored 4.0: overall = 0.80 × 100 = 80%

PERFORMANCE CLASSIFICATION:
  ≥ 85%  →  Top Performer
  ≥ 75%  →  Good Performer
  ≥ 65%  →  Satisfactory
  ≥ 50%  →  Needs Improvement
  < 50%  →  Critical

================================================================================
  4 INNOVATIVE FEATURES
================================================================================

FEATURE 1: PREDICTIVE PERFORMANCE ALERTS (TensorFlow.js)
  File: supervisor/predictive_alerts_new.php
  API:  api/innovative_features_api.php → get_all_trends

  WHAT IT DOES:
    Predicts each staff member's performance for the next year using
    a neural network trained on their historical KPI scores.

  HOW IT WORKS (TensorFlow.js in the browser):
    1. Fetches all staff yearly scores from the PHP API
    2. For each staff member with ≥3 years of data:
       a. Normalises year indices to [0,1] and scores to [0,1]
       b. Builds a Sequential Dense Neural Network:
          Input → Dense(8, ReLU) → Dense(4, ReLU) → Dense(1, Linear)
       c. Trains for 200 epochs using Adam optimiser, MSE loss
       d. Predicts next year's score
       e. Calculates confidence from MAE (Mean Absolute Error)
    3. Classifies risk based on predicted score:
       < 50%  → Critical
       < 65%  → High
       < 75%  → Medium
       ≥ 75%  → Low / Stable

  WHY NEURAL NETWORK OVER LINEAR REGRESSION?
    Linear regression can only model straight-line trends.
    A neural network with ReLU activation can model curves, plateaus,
    and recoveries — more realistic for human performance data.

  DISPLAY:
    - Threshold bar showing score ranges for each risk level
    - 4 summary cards with counts and percentages
    - Alert cards for Critical/High/Medium staff with recommendations
    - Collapsed section for Low/Stable staff (click to expand)
    - SweetAlert loading modal during training
    - Success toast when complete

FEATURE 2: SMART TRAINING RECOMMENDATIONS
  File: supervisor/training_recommendations.php
  Class: KPICalculator::getSmartTrainingRecommendations()

  WHAT IT DOES:
    Automatically matches each underperforming staff member to the
    most suitable training program based on their specific skill gaps.

  HOW IT WORKS:
    1. Identifies skill gaps by category:
       < 50% → Critical gap (priority 3)
       < 70% → Moderate gap (priority 2)
       < 85% → Minor gap   (priority 1)
    2. Compares gaps against 6 training programs, each with focus areas
    3. Calculates match score: higher score for lower performance in
       the program's focus area (more need = better match)
    4. Returns top 5 programs sorted by match score

  6 TRAINING PROGRAMS:
    1. Sales Excellence Program (4 weeks)
    2. Customer Service Mastery (3 weeks)
    3. Leadership Fundamentals (6 weeks, requires ≥85%)
    4. Operational Excellence (2 weeks)
    5. Time Management & Productivity (2 weeks)
    6. Advanced Product Knowledge (3 weeks)

FEATURE 3: ANOMALY DETECTION + AUTO-NARRATIVE INSIGHTS
  File: supervisor/anomaly_detection.php
  API:  api/innovative_features_api.php → detect_anomalies

  WHAT IT DOES:
    Scans a staff member's performance history for unusual patterns
    and generates plain-English explanations of what happened.

  ANOMALY TYPES DETECTED:
    Spike    — score increased by ≥5% in one year
    Drop     — score decreased by ≥5% in one year
    Decline  — score decreased for 3+ consecutive years

  AUTO-NARRATIVE:
    For each anomaly, the system generates a sentence explaining
    what happened and what it might mean for the supervisor.
    Example: "Performance dropped from 78% to 65% in 2024.
    Immediate attention and support recommended."

  DISPLAY:
    - Line chart with anomaly markers (Chart.js)
    - Anomaly cards with type, year, description
    - Narrative insights panel

FEATURE 4: ADVANCED PEER COMPARISON
  File: supervisor/peer_comparison_new.php
  API:  api/innovative_features_api.php → get_peer_comparison

  WHAT IT DOES:
    Compares a staff member's KPI performance against peers to
    identify where they stand relative to similar colleagues.

  COMPARISON MODES:
    Auto-select Best Match  — finds the most similar peer by score
    Top Performer           — compares against the highest scorer
    Team Average            — compares against team benchmark
    Specific Staff Member   — supervisor picks exact peer (NEW)

  HOW SIMILARITY IS CALCULATED:
    similarity = 100 - |staff_score - peer_score|
    Peers within ±10% are considered "similar".

  DISPLAY:
    - Side-by-side staff cards with VS badge (animated pulse)
    - Radar chart comparing KPI categories for both staff
    - Similarity score percentage
    - Insights panel highlighting key differences
    - "Similar Peers" section for quick comparison switching
