================================================================================
  SYSTEM PURPOSE
================================================================================

This system was built to solve a real workplace problem: supervisors in a
retail environment need a structured, data-driven way to monitor how their
sales assistants are performing across multiple areas of their job.

Without a system like this, supervisors would rely on spreadsheets or memory,
making it hard to spot trends, compare staff fairly, or justify training
decisions. This system replaces that with:

  - A structured 21-KPI scoring framework (1–5 scale, weighted)
  - Multi-year historical tracking (2022–2026)
  - Automatic performance classification and trend detection
  - AI-powered predictions using TensorFlow.js (neural network in browser)
  - Smart training recommendations matched to individual skill gaps
  - Anomaly detection to flag unusual performance changes
  - Peer comparison to benchmark staff against similar colleagues
  - Gamification to motivate staff through points and badges
  - Staff profile photos with professional headshot display
  - Accessibility-compliant HTML (aria-labels, scope, alt text)

The system is supervisor-only. There is no sales assistant login — supervisors
enter scores on behalf of their team and view all analytics.

  Chart.js 4.4.0      — Doughnut, line, bar, radar, histogram charts
  jQuery 3.7.1        — DOM manipulation and AJAX calls

  SweetAlert2 11.x    — Loading modals, confirmations, toast notifications
  Bootstrap Icons     — Icon library used throughout

BACKEND:
  PHP 7.4+            — Server-side scripting
  PDO                 — Prepared statements (SQL injection prevention)
  MySQL / MariaDB     — Relational database (XAMPP bundled)

ARCHITECTURE:
  3-tier: Presentation (supervisor/) → Business Logic (KPICalculator.php + api/)
          → Data Layer (MySQL via PDO)
  API endpoints return JSON, called by JavaScript fetch() / jQuery AJAX.

SECURITY:
  - PDO Prepared Statements  — prevents SQL injection
  - htmlspecialchars()       — prevents XSS on all output
  - Session Authentication   — requireLogin() on every protected page
  - requireLoginApi()        — returns HTTP 401 JSON for API endpoints
  - .htaccess Protection     — blocks direct access to config/ folder

================================================================================

TABLE: supervisors
  3 supervisor accounts: admin (John Anderson), manager1 (Sarah Williams),
  super1 (Michael Chen)
  Columns: supervisor_id, username, password, full_name, email, status,
           last_login, created_at, updated_at

TABLE: staff
  13 active sales assistants across Electronics, Fashion, Home Goods
  Columns: staff_id, staff_code, name, email, department, position,
           status, hire_date, photo, created_at
  photo column: stores filename only (e.g. "Alice_Johnson.jpg")

TABLE: kpi_master
  21 KPI definitions — reference/lookup table
  Columns: kpi_id, kpi_code, section, kpi_group, kpi_description,
           weight_percentage, section_number, display_order, created_at

TABLE: kpi_scores
  742 score rows across 2022–2026
  Columns: score_id, staff_id (FK), kpi_code (FK), evaluation_date,
           evaluation_year, score (1–5), weighted_score, created_at, updated_at
  Unique Key: (staff_id, kpi_code, evaluation_year)
  ON DUPLICATE KEY UPDATE — re-submitting same year updates, never duplicates

TABLE: staff_comments
  Supervisor feedback and training recommendations per staff per year
  Columns: comment_id, staff_id (FK), evaluation_year, supervisor_comment,
           training_recommendation, supervisor_id (FK), created_at, updated_at
  Unique Key: (staff_id, evaluation_year)

FOREIGN KEY RULES:
  kpi_scores → staff          CASCADE (delete staff → delete their scores)
  kpi_scores → kpi_master     RESTRICT (cannot delete KPI with scores)
  staff_comments → staff      CASCADE
  staff_comments → supervisors SET NULL (delete supervisor → comments kept)

================================================================================
  4 INNOVATIVE FEATURES
================================================================================

FEATURE 1: PREDICTIVE PERFORMANCE ALERTS (TensorFlow.js)
  File: supervisor/predictive_alerts_new.php

  Uses TensorFlow.js v4.17.0 — runs entirely in the browser.
  No Python, no server-side ML required.

  Model: Sequential Dense Neural Network
    Input → Dense(8, ReLU) → Dense(4, ReLU) → Dense(1, Linear)
  Training: Adam optimiser, MSE loss, 200 epochs
  Input: normalised year index [0,1], Output: predicted score [0,1]
  Confidence: 100 − MAE on training data, clamped 50–99%
  Minimum 3 years of data required per staff member

  Risk classification (matches threshold bar on page):
    < 50%  → Critical
    < 65%  → High
    < 75%  → Medium
    ≥ 75%  → Low / Stable

  Display:
    - Static threshold bar showing score ranges
    - 4 summary cards with counts + percentage of staff
    - Alert cards for Critical/High/Medium with recommendations
    - Collapsed green section for Low/Stable staff (click to expand)
    - SweetAlert loading modal during training + success toast when done
    - "Re-run Predictions" button retrains all models fresh

FEATURE 2: SMART TRAINING RECOMMENDATIONS
  File: supervisor/training_recommendations.php
  Class: KPICalculator::getSmartTrainingRecommendations()

  Identifies skill gaps by category score thresholds:
    < 50% → Critical gap (priority 3)
    < 70% → Moderate gap (priority 2)
    < 85% → Minor gap   (priority 1)

  Matches gaps to 6 training programs with compatibility scoring.
  Returns top 5 programs sorted by match score.

  Programs: Sales Excellence, Customer Service Mastery, Leadership
  Fundamentals, Operational Excellence, Time Management & Productivity,
  Advanced Product Knowledge

FEATURE 3: ANOMALY DETECTION + AUTO-NARRATIVE INSIGHTS
  File: supervisor/anomaly_detection.php

  Detects: Spike (≥5% increase), Drop (≥5% decrease),
           Consecutive Decline (3+ years decreasing)
  Generates plain-English narrative for each anomaly.
  Line chart with anomaly markers (Chart.js).

FEATURE 4: ADVANCED PEER COMPARISON
  File: supervisor/peer_comparison_new.php

  Compare modes:
    Auto-select Best Match — most similar peer by score
    Top Performer          — compare against highest scorer
    Team Average           — compare against team benchmark
    Specific Staff Member  — supervisor picks exact peer (NEW)

  Similarity = 100 − |staff_score − peer_score|
  Radar chart comparing KPI categories for both staff.
  Animated VS badge between the two staff cards.
